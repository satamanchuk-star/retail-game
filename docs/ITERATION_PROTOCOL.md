# Протокол итераций

Каждая итерация разработки должна иметь один главный результат.

## Порядок

1. Проверить текущее состояние репозитория.
2. Выбрать минимальный следующий слой по плану.
3. Реализовать без ломки существующих API.
4. Добавить или обновить тесты.
5. Запустить `ruff check .` и `pytest`.
6. Закоммитить изменения.
7. Создать PR через инструмент `make_pr`.

## Ближайшие слои

1. Пользователи и владение компаниями.
2. PostgreSQL persistence.
3. Транзакционное закрытие дня.
4. Партии товаров и срок годности.
5. Производство из сырья.
6. Склады, транспорт и маршруты.
7. Расширенные контракты и арбитраж.
