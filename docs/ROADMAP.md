# Дорожная карта: к интересной игре про цепочку поставок

Стратегия доведения по [TECHNICAL_SPEC.md](TECHNICAL_SPEC.md). Каждый пункт —
**вертикальный слой** (модель → баланс → движок → API → UI → тест) + прогон харнесса
баланса. Порядок = по убыванию ценности для «веселья», а не по технической чистоте.

Легенда: 🎯 ядро жанра · 🤖 одиночка-против-ботов · 🎨 читаемость · 🧰 инструмент · ⏸️ отложено.

## Фаза 0 — Целостность и инструмент (быстро, фундамент)

- [ ] **0.1 🧰 Привести доки к коду.** HANDOFF разжаловать в «исходное видение»; «сделано»
      только в DEVELOPMENT_STATUS. *(этот PR частично делает.)*
- [ ] **0.2 🧰 Харнесс баланса.** Расширить `simulation.py`: прогон мира на N дней без
      игрока, метрики — инфляция цен, число банкротств, доля лидера (HHI), доля
      out-of-stock, оборачиваемость. CLI/тест выводит «здоровье экономики». Это наш
      объективный «тест на баланс/веселье» для всех следующих фаз.
- [ ] **0.3 Унификация миров (опц., раньше или позже).** Свести «основную партию» к
      сессии по умолчанию, чтобы убрать дублирование global vs session. Можно отложить.

## Фаза 1 — 🎯 Наполнить цепочку (САМОЕ ЦЕННОЕ)

Без этого жанр пустой (сейчас производимы 2 товара из 30).

- [ ] **1.1 Сырьё и рецептуры.** Добавить типы сырья (мясо-сырьё, сахар, овощи-сырьё,
      зерно→мука→хлеб как 2 ступени и т.п.) и рецептуры так, чтобы **≈12–15 товаров**
      имели путь сырьё→производство. Часть товаров оставить «только закуп/импорт».
- [ ] **1.2 Сырьевой рынок.** Откуда брать сырьё: NPC-поставщики сырья с ценой/дефицитом
      (а не только стартовые остатки). Дефицит двигает себестоимость.
- [ ] **1.3 Узкие места.** Производство ограничено сырьём+мощностью+сроком; дефицит
      виден игроку (алерт). Прогнать харнесс: цепочка создаёт осмысленные решения.

## Фаза 2 — Оживить спрос (хук «ценовые войны»)

- [ ] **2.1 Эластичность.** Общий спрос реагирует на средний уровень цен (не только
      дележ доли). Калибровать харнессом, чтобы не было гипер-инфляции/коллапса.
- [ ] **2.2 Сегменты покупателей** (эконом/средний/премиум) + задействовать `income_index`
      региона. Демпинг и премиум становятся разными валидными стратегиями.
- [ ] **2.3 Out-of-stock → lost sales** уходят конкуренту; видно в отчёте. Замещение
      товаров — опционально.

## Фаза 3 — 🤖 Умные боты (это и есть одиночная игра)

- [ ] **3.1 Боты строят цепочку:** закупают сырьё, производят под спрос, расширяют
      мощности, а не только крутят `price_index`.
- [ ] **3.2 Реактивность:** реагируют на дефицит, цены конкурентов, сезон; меняют
      стратегию. Архетипы: осторожный, дискаунтер, премиум, логист-оптимизатор,
      производитель качества.
- [ ] **3.3 Сложность.** Параметр сложности ботов (для онбординга/хардкора).
- [ ] **3.4 Боты как контрагенты** (готовит Фазу 6): принимают/предлагают контракты.

## Фаза 4 — 🎯 Логистика (хук «оптимизация»)

- [ ] **4.1 Задействовать `logistics_risk`:** стоимость/потери/задержки доставки зависят
      от региона и расстояния.
- [ ] **4.2 Маршруты и cold chain:** скоропорт требует холодной цепи; нарушение → порча.
- [ ] **4.3 Склад как буфер + консолидация грузов.** Дистрибьютор получает реальную
      глубину и арбитраж (купил дёшево в регионе А → отвёз → продал в Б).

## Фаза 5 — 🎨 Игровой UI и онбординг (читаемость)

- [ ] **5.1 Туториал** первых ходов: цель, первое решение, первый результат.
- [ ] **5.2 Экраны ролей** вместо одной стены панелей; карта-центр.
- [ ] **5.3 Советник и алерты:** «затоваривание», «дефицит сырья», «конкурент демпингует»,
      «срок годности горит». Перевод цифр в смысл.
- [ ] **5.4 Графики/фильтры** для рынка, цен, финансов.

## Фаза 6 — Контракты и торговля (хук «торговля»)

- [ ] **6.1 Регулярные поставки, отсрочка платежа, SLA, частичное исполнение.**
- [ ] **6.2 Репутация и последствия** срыва; простой арбитраж/штрафы.
- [ ] **6.3 NPC-переговоры:** боты предлагают/торгуются по условиям.

## Фаза 7 — События и сезонность (живой мир)

- [ ] **7.1 Конкретные события:** жара/мороз (спрос категорий), праздники, скачки
      топлива (логистика), урожайность (сырьё), проверки.
- [ ] **7.2 Прогноз** событий (игрок может готовиться) + случайные шоки.

## Фаза 8 — Рост империи и скоринг (хук «tycoon»)

- [ ] **8.1 Осмысленный скоринг** (капитал + доля рынка + устойчивость), недельные цели.
- [ ] **8.2 Экспансия по регионам**, прогресс-вехи, престиж/реиграбельность.

## Отложено (⏸️ после «весёлой» одиночки)

- ⏸️ Полноценный мультиплеер: лобби/UI сессий, чат, переговоры между игроками,
  синхронизация, модерация.
- ⏸️ Нормализация PostgreSQL + транзакционный день (нужно для мультиплеера, не для
  одиночки).
- ⏸️ Админка баланса в UI (пока — `balance.py` + харнесс).
- ⏸️ Рефакторинг `engine.py` god-class в модули подсистем (делать, когда начнёт мешать).

## Рекомендуемый ближайший порядок

**0.1 → 0.2 → 1.1 → 1.2 → 2.1 → 3.1**

То есть: убрать враньё в доках, построить измеритель баланса, **наполнить цепочку**,
дать сырьевой рынок, включить эластичность, научить ботов строить цепочку. После этого
у нас впервые появляется *настоящая игра про цепочку поставок против умного соперника* —
дальше Фазы 4–8 углубляют и полируют.
