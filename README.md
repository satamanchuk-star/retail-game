# Цепочка прибыли

Браузерный прототип экономической стратегии про ритейл, производство и дистрибуцию в едином живом рынке.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Откройте `http://127.0.0.1:8000`.

## Что уже есть

- FastAPI-приложение с `lifespan`.
- HTML/CSS/JS-интерфейс в стиле серьёзной tycoon-стратегии.
- 5 регионов с русским уклоном.
- Полная стартовая FMCG-матрица из 30 товарных позиций.
- Первый расчёт игрового дня для ритейлера.
- API состояния мира и симуляции дня.
- Базовые тесты доменной логики и API.


## Следующий слой прототипа

- `POST /api/companies` создаёт компанию игрока в одной из трёх ролей.
- `POST /api/decisions/{company_id}` сохраняет дневные решения: цену, производство, логистику и маркетинг.
- `POST /api/contracts` создаёт формальный контракт поставки или логистики.
- `POST /api/close-day` закрывает день всего рынка и возвращает отчёты по компаниям.
- `GET /api/ratings` показывает общий и ролевой рейтинг компаний.
- `GET /api/banks` и `POST /api/loans` добавляют NPC-банки и кредиты с ежедневными процентами.
- Интерфейс позволяет создать простой контракт поставки и видеть его статус.
- Старый endpoint `POST /api/simulate-day/{company_id}` сохранён для совместимости.


## Быстро увидеть результат

1. Запустите приложение.
2. Нажмите `Запустить демо 7 дней`.
3. Интерфейс покажет график прибыли, таблицу операций, отчёты компаний, кредиты, контракты и новости рынка.

## Сохранение состояния прототипа

По умолчанию мир хранится в памяти процесса, чтобы тесты и локальные демо всегда стартовали чисто.
Для сохранения прогресса между перезапусками задайте путь к JSON-снапшоту:

```bash
PROFIT_CHAIN_STATE_FILE=.data/game_state.json uvicorn app.main:app --reload
```

- `GET /api/persistence` показывает, включено ли сохранение, путь к снапшоту и счётчики мира.
- Все мутирующие операции API сохраняют состояние автоматически, если `PROFIT_CHAIN_STATE_FILE` задан.
- Это промежуточный слой прототипа; финальный multiplayer-фундамент планируется переводить на PostgreSQL.

## Пользователи и владение компаниями

- `POST /api/auth/register` регистрирует игрока и возвращает bearer-токен.
- `POST /api/auth/login` выдаёт новый bearer-токен существующему игроку.
- `GET /api/me` возвращает текущего игрока по заголовку `Authorization: Bearer <token>`.
- Если компания создана с bearer-токеном, она получает `owner_user_id`.
- Управление защищённой компанией доступно только её владельцу.
- `GET /api/state` отдаёт публичное состояние без `users`, `sessions` и `password_hash`.

## Статус разработки

- `GET /api/project/status` возвращает текущую стадию проекта, процент закрытых крупных этапов и дорожную карту.
- Главный экран показывает блок `Статус разработки`, чтобы результат каждой итерации был виден без чтения переписки.


## Финансовый контур

- `GET /api/finances` — P&L/cash-flow/НДС/долговая нагрузка по всем компаниям.
- `GET /api/finances/{company_id}` — финансовый отчёт одной компании с последними проводками.

## Сырьё и рецептуры

- `GET /api/state` отдаёт `raw_materials`, `production_recipes`, `raw_inventories` и `assets`.
- Производитель больше не выпускает товар из воздуха: закрытие дня списывает сырьё по рецептуре.
- Если сырья не хватает, фактический выпуск ограничивается, а журнал закрытия дня получает операцию `raw_material_shortage`.
- Главный экран показывает блок `Сырьё и рецептуры`, чтобы видеть рецепты и остатки производителя.

## Операционные объекты

- `GET /api/assets` возвращает магазины, заводы и склады компаний.
- Объекты задают дневные мощности ролей и постоянные расходы: магазин ограничивает продажи, завод — выпуск, склад — логистическую мощность.
- Главный экран показывает блок `Операционные объекты`, чтобы результат был виден без просмотра JSON.


## DB-снапшот мира

Для проверки durable-хранилища можно включить SQLAlchemy async snapshot через переменную окружения:

```bash
PROFIT_CHAIN_DATABASE_URL=sqlite+aiosqlite:///.data/game_state.db python -m uvicorn app.main:app --reload
```

Для PostgreSQL используйте URL формата `postgresql+asyncpg://user:password@host:5432/dbname` через переменные окружения, не коммитьте секреты в git.

- `GET /api/database/status` показывает, включён ли DB-режим, активен ли engine и какой диалект используется. Строка подключения наружу не отдаётся.
